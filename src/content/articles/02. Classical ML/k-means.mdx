# K-Means Clustering

Das **K-Means Clustering** ist einer der populärsten Algorithmen im Bereich des **Unsupervised Learning** (Unüberwachtes Lernen). Während Klassifikatoren wie SVM oder k-NN wissen müssen, welche Punkte zu welcher Gruppe gehören, findet K-Means diese Gruppen ganz von selbst. Er erkennt Muster und Strukturen in Daten, die auf den ersten Blick wie eine ungeordnete Punktwolke aussehen.

---

## Intuition: Ordnung finden ohne Vorwissen

:::beginner
Stell dir vor, du schüttest eine Kiste bunter Bausteine auf den Boden. Du kennst die Farben nicht, aber du willst sie in **3 Häufen** sortieren. 
1. Du legst drei Platzhalter (Zentroiden) irgendwo in den Raum.
2. Du nimmst jeden Stein und legst ihn zu dem Platzhalter, der am nächsten liegt.
3. Sobald alle Steine verteilt sind, schiebst du jeden Platzhalter genau in die Mitte seines neuen Haufens.
4. Das wiederholst du so lange, bis sich die Platzhalter nicht mehr bewegen.
:::

:::medium
K-Means ist ein iterativer Algorithmus, der Datenpunkte in $k$ Cluster unterteilt. Ziel ist es, die Varianz innerhalb der Cluster zu minimieren. 
Der Prozess besteht aus zwei Hauptschritten:
1.  **Zuweisung:** Jeder Datenpunkt wird dem nächstgelegenen Zentroiden (Mittelpunkt) zugewiesen.
2.  **Update:** Die Position jedes Zentroiden wird als Mittelwert aller ihm zugewiesenen Punkte neu berechnet.

Dieser Zyklus wiederholt sich, bis die Zentroiden stabil bleiben oder eine maximale Anzahl an Iterationen erreicht ist.
:::

:::advanced
Mathematisch minimiert K-Means die Summe der quadrierten Abstände (Within-Cluster Sum of Squares, WCSS):

$$
\text{WCSS} = \sum_{j=1}^{k} \sum_{x \in S_j} \| x - \mu_j \|^2
$$

Dabei ist $S_j$ die Menge der Punkte im Cluster $j$ und $\mu_j$ der Zentroid dieses Clusters. Ein bekanntes Problem ist, dass der Algorithmus in einem lokalen Minimum stecken bleiben kann, abhängig von der initialen (zufälligen) Position der Zentroiden. Moderne Varianten wie **K-Means++** nutzen eine klügere Initialisierung, um dieses Risiko zu minimieren.
:::

## Herausforderungen

- **Wahl von k:** Wie viele Cluster gibt es wirklich? Man nutzt oft die **Ellbogen-Methode**, um das optimale $k$ zu finden.
- **Ausreißer:** Einzelne Punkte, die sehr weit weg liegen, können die Zentroiden stark verfälschen.
- **Nicht-sphärische Formen:** K-Means funktioniert am besten bei kreisförmigen (sphärischen) Clustern. Bei länglichen oder mondförmigen Clustern scheitert er oft (hier wären Algorithmen wie DBSCAN besser).

## Implementierung in Python

K-Means ist ideal, um in unbekannten Daten Gruppen zu entdecken.

```python
from sklearn.cluster import KMeans
import numpy as np

# 1. Daten vorbereiten (X: Features, keine Labels y!)
X = np.array([[1, 2], [1, 4], [1, 0], [10, 2], [10, 4], [10, 0]])

# 2. Modell erstellen (n_clusters = k)
kmeans = KMeans(n_clusters=2, random_state=0)
kmeans.fit(X)

# 3. Ergebnisse abfragen
print(f"Cluster-Zugehörigkeit: {kmeans.labels_}")
print(f"Zentroiden (Mittelpunkte): \n{kmeans.cluster_centers_}")

# 4. Einen neuen Punkt zuweisen
print(f"Neuer Punkt (0,0) gehört zu Cluster: {kmeans.predict([[0, 0]])[0]}")
```

**Erklärung:**
- `n_clusters`: Hier legst du fest, in wie viele Gruppen die Daten geteilt werden sollen.
- `cluster_centers_`: Dies sind die Koordinaten der "Platzhalter" (Zentroiden), die am Ende im Zentrum ihrer jeweiligen Gruppe liegen.

## Zusammenfassung

- **Unüberwacht:** Benötigt keine Labels (Klassen) zum Lernen.
- **Zentroiden-basiert:** Findet die Mittelpunkte von Datenwolken.
- **Einsatz:** Kundensegmentierung, Bildkompression (Farbreduktion) und Anomalieerkennung.
